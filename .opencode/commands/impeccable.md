Use the impeccable skill to handle this frontend design task. Load the skill first, then follow its instructions precisely.

The user's request is: $ARGUMENTS

Steps:
1. Load the impeccable skill to get full instructions
2. Follow the skill's setup gates (context, product, command reference)
3. Run `node .agents/skills/impeccable/scripts/load-context.mjs` to load PRODUCT.md and DESIGN.md
4. Apply the matching register (brand or product) and shared design laws
5. Execute the requested command or general design task

Available sub-commands:
- craft [feature] - Shape then build end-to-end
- shape [feature] - Plan UX/UI before writing code
- teach - Set up PRODUCT.md and DESIGN.md context
- document - Generate DESIGN.md from existing code
- extract [target] - Pull reusable tokens and components
- critique [target] - UX design review with scoring
- audit [target] - Technical quality checks (a11y, perf, responsive)
- polish [target] - Final quality pass before shipping
- bolder [target] - Amplify safe or bland designs
- quieter [target] - Tone down aggressive designs
- distill [target] - Strip to essence, remove complexity
- harden [target] - Production-ready: errors, i18n, edge cases
- onboard [target] - Design first-run flows, empty states
- animate [target] - Add purposeful animations and motion
- colorize [target] - Add strategic color
- typeset [target] - Improve typography hierarchy
- layout [target] - Fix spacing, rhythm, visual hierarchy
- delight [target] - Add personality and memorable touches
- overdrive [target] - Push past conventional limits
- clarify [target] - Improve UX copy, labels, error messages
- adapt [target] - Adapt for different devices and screen sizes
- optimize [target] - Diagnose and fix UI performance
- live - Interactive live variant mode (requires dev server)
